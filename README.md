# 🧪 OrangeHRM Manual Testing Project

## 📌 Project Overview

This project involves comprehensive **Manual Testing** of the **OrangeHRM Web Application**, focusing on the **Login Module** and **Leave Management Module**.

The objective of this project was to validate application functionality, verify business requirements, identify defects, and ensure a smooth user experience through structured software testing practices.

The project was executed following industry-standard QA processes including **Requirement Analysis, Test Planning, Test Case Design, Test Execution, Defect Reporting, and Requirement Traceability Matrix (RTM) creation**.

## 🐛 Major Defects Identified

### D012 – Leave Summary Access Issue

Unauthorized users were able to access Leave Summary information.

### D014 – Holiday Validation Issue

System accepted invalid holiday configurations without proper validation.

### D015 – Duplicate Leave Type Issue

System allowed creation of duplicate leave types.

### D017 – Assign Leave Validation Issue

Leave assignment validation was not triggered correctly for invalid inputs.

### D021 – Duplicate Leave Request Issue

System allowed submission of duplicate leave requests.

### D024 – Leave Calendar Display Issue

Leave Calendar data was not displayed correctly under specific scenarios.
